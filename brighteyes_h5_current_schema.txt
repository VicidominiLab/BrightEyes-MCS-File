BrightEyes MCS HDF5 Current Schema - Version 0.0.6
==================================================

Status
------

Version 0.0.6 is the current calibrated-output schema used by
brighteyes_mcs_file. It intentionally breaks older root-level BrightEyes
layouts. A writer emits three semantic root groups, plus an optional root
thumbnail dataset:

  /raw
  /calibration
  /output
  /thumbnail              optional dataset copied from the input file

Legacy input payloads are renamed into stable raw paths:

  old /data                 -> /raw/spad
  old /data_channels_extra  -> /raw/aux
  old /data_analog          -> /raw/analog, if present
  old /thumbnail            -> /thumbnail, if present

Legacy configuration groups are preserved verbatim under /raw/legacy. New
readers should not parse /raw/legacy for routine analysis. They should read the
normalized metadata and axes exposed by /raw, /calibration, and /output.


Root
----

/.attrs:
  data_format_version = "0.0.6"
  schema_name = "brighteyes_mcs_file"
  schema_variant = "default"
  file_kind = "calibrated_measurement"
  default = "/raw/spad"
  data_path = "/raw"
  metadata_path = "/raw/metadata"
  axes_path = "/raw/axes"
  legacy_path = "/raw/legacy"
  calibration_path = "/calibration"
  output_path = "/output"
  contains_measured_payload = True
  contains_legacy_configuration = True
  contains_calibration = True
  contains_output = True | False
  compatibility_root_configuration_groups = False
  comment = "<source root comment or user supplied comment>"

Root rule:
  Do not write /metadata, /axes, /legacy, /original_data, /configurationGUI,
  /configurationFPGA, /configurationGUI_beforeStart, or
  /configurationSpadFCSmanager at the root in 0.0.6 files.

Optional path rule:
  Attributes that point to optional groups or datasets use an empty string when
  the target is absent. Optional groups and datasets are omitted when absent.
  Required path attributes must point to an existing object; readers should
  treat a missing required target as a schema error.

Output absence rule:
  If contains_output = False, output_path may still reserve "/output", but the
  /output group may be absent. Readers should check contains_output and the
  existence of /output before loading derived products.


/raw
----

Purpose:
  Measured payload values, normalized acquisition metadata, physical axes, and
  the verbatim legacy source configuration.

/raw.attrs:
  description = "Measured BrightEyes-MCS payload with normalized metadata, axes, and legacy configuration."
  source_file = "<input HDF5 path>"
  source_data_format_version = "<input /.attrs.data_format_version or unknown>"
  source_root_default = "<input /.attrs.default, usually data>"
  source_payload_paths_json = "[\"/data\", \"/data_channels_extra\", ...]"
  values_preserved = True
  shape_preserved = True
  dtype_preserved = True
  spad = "/raw/spad"
  aux = "/raw/aux"
  analog = "/raw/analog"
  thumbnail = "/thumbnail"
  metadata_path = "/raw/metadata"
  axes_path = "/raw/axes"
  legacy_path = "/raw/legacy"
  source_axis_order = "repetition,z,y,x,time_bin,channel"

Children:
  /raw/spad                 required
  /raw/aux                  optional, copied from old /data_channels_extra
  /raw/analog               optional, copied from old /data_analog
  /raw/metadata             required
  /raw/axes                 required
  /raw/legacy               required


/raw/spad
---------

Contents:
  Primary digital FIFO photon-count histograms copied from old /data.

Shape:
  Raster:
    (repetition, z, y, x, time_bin, detector_channel)

  Circular/subpixel:
    (repetition, z, y, x, circular_repetition, circular_point,
     time_bin, detector_channel)

Attrs:
  data_role = "spad_detector_counts"
  source_input_path = "/data"
  actual_source_path = "<actual source dataset path copied>"
  source_fifo = "FIFO"
  source_key = "data"
  units = "counts"
  axis_order = "repetition,z,y,x,time_bin,detector_channel"
  time_axis_path = "/raw/axes/digital_time_ns"
  channel_axis_path = "/raw/axes/detector_channel_index"
  metadata_path = "/raw/metadata"
  legacy_metadata_path = "/raw/legacy"
  calibration_result_path = "/calibration/results/spad"
  shape_json = "[...]"
  dtype_preserved_from_source = True

/raw/analog
-----------

Contents:
  Optional analog FIFO samples copied from old /data_analog.

Shape:
  Same non-channel dimensions as /raw/spad when produced by the same
  acquisition, with final axis analog_channel.

Attrs:
  data_role = "analog_fifo"
  source_input_path = "/data_analog"
  actual_source_path = "<actual source dataset path copied>"
  source_fifo = "FIFOAnalog"
  source_key = "data_analog"
  units = "adc_counts"
  adc_calibration_formula = "voltage_v = adc_offset_v + adc_slope_v_per_adc_unit * adc_counts"
  adc_offset_v = <float or NaN>
  adc_slope_v_per_adc_unit = <float or NaN>
  adc_channel_offset_v_json = "[NaN, NaN, ...]"
  adc_channel_slope_v_per_adc_unit_json = "[NaN, NaN, ...]"
  axis_order = "repetition,z,y,x,time_bin,analog_channel"
  time_axis_path = "/raw/axes/analog_time_ns"
  channel_axis_path = "/raw/axes/analog_channel_index"
  metadata_path = "/raw/metadata"
  legacy_metadata_path = "/raw/legacy"
  calibration_result_path = ""
  shape_json = "[...]"
  dtype_preserved_from_source = True

  ADC calibration:
    Analog payload values are stored as raw ADC counts by default. The scalar
    offset/slope attrs are NaN until calibrated. Per-channel JSON attrs have
    one value per analog channel; common two-channel analog payloads therefore
    default to two NaN values.

/raw/axes
---------

/raw/axes.attrs:
  description = "Physical and logical axes for measured payload and derived products."

Datasets:

  /raw/axes/repetition_index
    shape = (nrep,), dtype = int64
    attrs: axis = "repetition", units = "index"

  /raw/axes/x_um
  /raw/axes/y_um
  /raw/axes/z_um
    dtype = float64
    values = linspace(offset_um - range_um/2, offset_um + range_um/2, n)
    attrs:
      axis = "x" | "y" | "z"
      units = "um" | "index"
      range_um = <range along this axis>
      offset_um = <offset along this axis>
      pixel_size_um = <spacing between adjacent axis entries>
      coordinate_rule = "linspace(offset_um - range_um/2, offset_um + range_um/2, n)"

  /raw/axes/digital_time_ns
    shape = (time_bins,), dtype = float64
    attrs:
      axis = "time_bin"
      units = "ns" | "index"
      acquisition_mode = "normal" | "dfd"
      timing_reference = "pixel_dwell" | "laser_period"
      time_bins = <int>
      time_bins_meaning = "<human-readable meaning>"
      bin_width_ns = <float or NaN>

  /raw/axes/analog_time_ns
    optional, present when /raw/analog exists.

  /raw/axes/detector_channel_index
  /raw/axes/aux_channel_index
  /raw/axes/analog_channel_index
    channel index axes, present when the corresponding payload exists.

/output
-------

Purpose:
  Derived analysis products. Output runs are independent enough for readers to
  inspect /output/<run_id>/metadata, /output/<run_id>/axes, and products without
  parsing /raw/legacy.

  Top-level /output children are the output inventory. The collection can
  contain /output/virtual_channels, image output run groups, and trace output
  run groups. Readers should use /output.keys() to list available outputs,
  then inspect each child group's attrs, metadata, axes, and products. Readers
  should identify image runs by output_type = "image_tool" on a group and
  trace products by output_type = "trace" on
  /output/<trace_run_id>/products/trace.

/output.attrs:
  default = "sum_channels_001"
  default_irf_trace_id = "sum_irf_001"
  default_ref_trace_id = "sum_ref_001"

  Default run selection:
    /output.attrs is a small index only. If /output/sum_channels_001 is
    present, default points to that channel-skew-corrected aligned run. The
    uncorrected sum remains available as
    /output/sum_channels_without_corrections_001. If a default target is absent,
    the corresponding attr is an empty string.

Virtual channel outputs:
  Purpose:
    Typed HDF5 virtual datasets exposing one source channel without copying raw
    data.
  /output/virtual_channels
    attrs:
      description = "Typed virtual datasets mapping individual source channels."
      spad_source_data_path = "/raw/spad or empty string"
      aux_source_data_path = "/raw/aux or empty string"
      analog_source_data_path = "/raw/analog or empty string"
      spad_channel_count = <int or 0>
      aux_channel_count = <int or 0>
      analog_channel_count = <int or 0>
      source_axis_order = "<source axis order for first available source>"
      virtual_axis_order = "<channel-dropped axis order for first available source>"
      naming_rule = "<kind>/channel_<channel_index>"
      kind_groups_json = "[\"spad\", \"aux\", \"analog\"]"

  /output/virtual_channels/spad
  /output/virtual_channels/aux
  /output/virtual_channels/analog
    Optional typed groups, present only when the corresponding source payload
    exists. Group attrs include:
      virtual_channel_type = "spad" | "aux" | "analog"
      source_data_path = "/raw/spad" | "/raw/aux" | "/raw/analog"
      source_channel_axis = -1
      source_channel_axis_path = "/raw/axes/<channel axis>"
      channel_count = <int>
      source_axis_order
      virtual_axis_order
      data_layout = "raster_6d" | "circular_8d"
      units = "counts" | "adc_counts"
      naming_rule = "channel_<channel_index>"

  /output/virtual_channels/<kind>/channel_<index>
    Virtual dataset with shape equal to the source payload shape without the
    final channel axis. Dataset attrs include:
      virtual_channel_name = "channel_<index>"
      virtual_channel_type = "spad" | "aux" | "analog"
      virtual_source_file = "."
      source_data_path = "/raw/spad" | "/raw/aux" | "/raw/analog"
      source_channel_index = <int>
      source_channel_axis = -1
      source_channel_axis_path = "/raw/axes/<channel axis>"
      source_selection = "[:, ..., <index>]"
      axis_order = "<channel-dropped axis order>"
      units = "counts" | "adc_counts"
      is_virtual_dataset = True

  /output/<run_id>/intermediates/<name>
    Optional intermediate datasets needed to interpret the output. APR outputs
    store shift vectors at:
      /output/<apr_run_id>/intermediates/shift_vectors

      axes = "detector_channel,shift_component"
      shift_components_json = "[\"dy\", \"dx\"]"
    APR run attrs and /output/<apr_run_id>/metadata.attrs include:
      shift_vectors_path = "/output/<apr_run_id>/intermediates/shift_vectors"

Writer helper behavior:
  write_h5_output_run(...) appends one /output/<run_id> analysis run by
  default. If the requested run id exists, it writes the next numbered id such
  as apr_002 unless output_key_overwrite=True, in which case only the requested
  /output/<run_id> child is replaced. mode="copy" writes to a copied source
  file. mode="outputs_only" writes a self-describing HDF5 fragment containing
  root attrs plus /output, without /raw or /calibration.

Reader behavior
---------------

New readers should prefer:

  /raw/spad
  /raw/aux
  /raw/metadata
  /raw/axes
  /calibration/metadata
  /calibration/axes
  /calibration/results/<product>
  /output/<run_id>/metadata
  /output/<run_id>/axes
  /output/<run_id>/products/*

Use /raw/legacy only for audit, debugging, or fields not yet normalized. The
normalized metadata deliberately uses readable names with units in the key
suffix: *_um, *_us, *_ns, *_mhz, and *_s.
