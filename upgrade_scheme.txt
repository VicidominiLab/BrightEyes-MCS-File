BrightEyes-MCS calibrated HDF5 upgrade scheme proposal
======================================================

Status
------

Draft proposal only. This file describes the target output structure to review
before changing h5_data_calibrator.py.

Scheme version:
  0.0.5

Main requested change
---------------------

The current calibrator copies the input HDF5 file and appends /calibration and
optionally /output. The upgraded writer should instead create a new HDF5 file
from scratch with three required root groups and one optional root preview
dataset:

  /raw
  /calibration
  /output
  /thumbnail       optional dataset

The measured input payload is stored directly under /raw:

  /raw/spad
    old input /data

  /raw/aux
    old input /data_channels_extra

Optional payloads, normalized metadata, axes, and legacy configuration are also
stored below /raw. This breaks root-level compatibility intentionally; old
configuration* groups are preserved under /raw/legacy.


Top-level layout
----------------

Proposed tree:

/
  attrs:
    data_format_version = "0.0.5"
    schema_name = "brighteyes_mcs_file"
    schema_variant = "raw_root_group_root_thumbnail"
    file_kind = "calibrated_measurement"
    default = "/raw/spad"
    data_path = "/raw"
    metadata_path = "/raw/metadata"
    axes_path = "/raw/axes"
    legacy_path = "/raw/legacy"
    calibration_path = "/calibration"
    output_path = "/output"
    contains_measured_payload = True
    contains_legacy_configuration = True
    contains_calibration = True | False
    contains_output = True | False
    compatibility_root_configuration_groups = False
    comment = "<source root comment or user supplied comment>"

  raw/
    # Original measured payload, normalized metadata, axes, and legacy source
    # configuration. This is the only root raw payload container.

  calibration/
    # Calibration settings, provenance, histograms, fit results, delays, and
    # channel-skew results.

  output/
    # Optional derived analysis outputs such as channel-summed images.

  thumbnail
    # Optional preview dataset copied from old /thumbnail.

Root rule:
  No /metadata, /axes, /legacy, /original_data, /configurationGUI,
  /configurationFPGA, /configurationGUI_beforeStart, or
  /configurationSpadFCSmanager groups are written at the root.


/raw
-----

Purpose:
  Store the input measured payload in canonical names and provide all metadata
  needed to interpret it. /raw is both the payload container and the place
  where old configuration is preserved.

/raw.attrs:
  description = "Measured BrightEyes-MCS payload with normalized metadata, axes, and legacy configuration."
  source_file = "<input HDF5 path>"
  source_data_format_version = "<input /.attrs.data_format_version or unknown>"
  source_root_default = "<input /.attrs.default, usually data>"
  source_payload_paths_json = "[\"/data\", \"/data_channels_extra\", ...]"
  values_preserved = True
  shape_preserved = True
  dtype_preserved = True
  spad = "/raw/spad"
  aux = "/raw/aux"
  analog = "/raw/analog"
  thumbnail = "/thumbnail"
  metadata_path = "/raw/metadata"
  axes_path = "/raw/axes"
  legacy_path = "/raw/legacy"
  source_axis_order = "repetition,z,y,x,time_bin,channel"
  note = "Dataset attrs are added for readability; numeric payload values are unchanged."

Recommended children:

  /raw/spad
  /raw/aux
  /raw/analog                 optional
  /raw/metadata
  /raw/axes
  /raw/legacy


/raw/spad
-------------

Source path:
  old input /data

Meaning:
  Primary digital FIFO photon-count histograms.

Shape:
  (R, Z, Y, X, Td, C) for current BrightEyes-MCS files.

Dtype:
  uint16 in normal observed files.

Required attrs:
  data_role = "spad_detector_counts"
  long_name = "SPAD digital detector photon counts"
  description = "Photon counts decoded from the BrightEyes-MCS digital FIFO."
  source_input_path = "/data"
  source_fifo = "FIFO"
  source_key = "data"
  units = "counts"
  axis_order = "repetition,z,y,x,time_bin,detector_channel"
  axis_0 = "repetition"
  axis_1 = "z"
  axis_2 = "y"
  axis_3 = "x"
  axis_4 = "time_bin"
  axis_5 = "detector_channel"
  time_axis_index = 4
  channel_axis_index = 5
  time_axis_path = "/raw/axes/digital_time_ns"
  channel_axis_path = "/raw/axes/detector_channel_index"
  metadata_path = "/raw/metadata"
  legacy_metadata_path = "/raw/legacy"
  calibration_result_path = "/calibration/results/spad"
  expected_shape_source = "configurationFPGA/configurationSpadFCSmanager plus dataset shape"
  shape_json = "[R, Z, Y, X, Td, C]"
  dtype_preserved_from_source = True

/raw/legacy
------------

Purpose:
  Preserve the old configuration scheme inside /raw. These groups are for
  traceability, not for normal analysis.

/raw/legacy.attrs:
  description = "Verbatim legacy BrightEyes-MCS root attributes and configuration groups."
  source_file = "<input HDF5 path>"
  source_group_paths_json = "[\"/configurationGUI\", \"/configurationFPGA\", ...]"
  use_for_analysis = "Prefer /raw/metadata for normalized analysis metadata."

Groups copied from the input file:

/raw/legacy/root_attrs.attrs:
  <all input root attrs>

/raw/legacy/configurationGUI.attrs:
  <all attrs copied from input /configurationGUI>

/raw/legacy/configurationGUI_beforeStart.attrs:
  <all attrs copied from input /configurationGUI_beforeStart, if present>

/raw/legacy/configurationFPGA.attrs:
  <all attrs copied from input /configurationFPGA>

/raw/legacy/configurationSpadFCSmanager.attrs:
  <all attrs copied from input /configurationSpadFCSmanager, if present>

/raw/legacy/rawStreamAcquisition.attrs:
  <all attrs copied from input /rawStreamAcquisition, if present>

Legacy preservation rule:
  Do not normalize names inside /raw/legacy copied groups. Keep original attr
  names and values there. Put normalized names only under /raw/metadata.

/output
-------

Purpose:
  Keep derived products separate from measured payload and calibration. The
  current /output/<run_id> structure is acceptable, but source pointers should
  use the upgraded /raw paths.

/output.attrs:
  description = "Derived analysis outputs. Measured payload is stored under /raw."
  source_data_path = "/raw/spad"
  source_aux_data_path = "/raw/aux or empty string"
  metadata_path = "/raw/metadata"
  axes_path = "/raw/axes"
  legacy_path = "/raw/legacy"

Each /output/<run_id>.attrs should include:
  source_data_path = "/raw/spad"
  source_aux_data_path = "/raw/aux or empty string"
  source_calibration_path = "/calibration/results/spad"
  source_aux_calibration_path = "/calibration/results/aux or empty string"
  source_metadata_path = "/raw/metadata"
  source_axes_path = "/raw/axes"
  source_legacy_path = "/raw/legacy"
  input_axis_order = "repetition,z,y,x,time_bin,detector_channel"
  output_axis_order = "<output axes>"
  time_axis_path = "/output/<run_id>/axes/time_ns"


Implementation rules for h5_data_calibrator.py
----------------------------------------------

1. Replace the copy-then-append behavior for the upgraded mode.
   The writer should create a new empty HDF5 output file and explicitly write
   only /raw, /calibration, and /output at the root.

2. Preserve user data and avoid accidental data mutation.
   Numeric values under /raw/spad, /raw/aux, and optional /raw/analog
   must match the corresponding input datasets exactly. Copy dtype, shape, and
   preferably chunks/compression when practical.

3. Break old root-level compatibility intentionally.
   Do not write old configuration groups at the root. Preserve them under
   /raw/legacy.

4. Add descriptive attrs to payload datasets.
   The current input files usually have no attrs on /data and
   /data_channels_extra. The upgraded file should add axis, units, source path,
   metadata path, and calibration path attrs.

5. Keep legacy configuration exact.
   Copy old configuration attrs into /raw/legacy without renaming them.
   Normalized names belong under /raw/metadata.

6. Use JSON strings for structured attrs.
   Attribute names storing JSON should end with _json.

7. Calibration and output should reference canonical paths.
   New calibration/output source attrs should point to /raw/spad and
   /raw/aux.

8. Missing optional source objects should be explicit.
   If /data_channels_extra, /data_analog, /thumbnail, or
   /rawStreamAcquisition are absent, omit their datasets/groups and set the
   corresponding metadata paths to an empty string.

9. Record conflicts instead of guessing silently.
   If metadata-derived dimensions disagree with dataset shapes, use dataset
   shape as authoritative for payload dimensions and append a warning to
   /raw/metadata.attrs.metadata_warnings_json.


Minimal conversion order
------------------------

1. Open source data file read-only.
2. Open output file as a new file.
3. Write root attrs.
4. Create /raw, /calibration, and /output as the required root groups.
5. Copy old /data to /raw/spad and old /data_channels_extra to /raw/aux.
6. Copy optional old /data_analog to /raw/analog and /thumbnail to /thumbnail.
7. Copy root attrs and legacy configuration groups to /raw/legacy.
8. Build /raw/metadata normalized attrs and acquisition subgroups.
9. Build /raw/axes datasets.
10. Run calibration using /raw/spad and /raw/aux as source datasets.
11. Build optional /output products using /raw and /raw/metadata paths.


Open questions before implementation
------------------------------------

1. Should /output always be created, even when create_output=False, so the root
   always has exactly /raw, /calibration, /output?

2. Should /calibration always be created for uncalibrated conversions, or only
   when calibration is actually performed?

3. Should the canonical /raw paths keep the current 6D layout, or should a
   future version split circular scan dimensions into explicit axes?

4. Should /calibration/provenance/input_data_metadata remain as a compatibility
   snapshot, or should calibration provenance point only to /raw/metadata?
