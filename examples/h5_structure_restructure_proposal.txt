BrightEyes calibrated HDF5 schema v0.0.5
========================================

Status:
  Proposed clean output schema for calibrate_h5_file.

Intent:
  A calibrated BrightEyes FLIM HDF5 file should contain the measurement data,
  normalized human-readable metadata, raw instrument metadata for traceability,
  and calibration outputs in one predictable structure.

Mode selected:
  Clean v0.0.5 mode.

Important clean-mode decision:
  /data is a group. Reader code should use /.attrs.default or
  /metadata.attrs.data_primary_path to find the main data.

Comment syntax in this document:
  Lines after "#" are explanatory comments for humans and implementers. HDF5
  itself does not store comments; the writer should store the corresponding
  information as attrs and datasets.


Top-level tree
--------------

/
  .attrs:
    data_format_version = "0.0.5"
      # Required. File schema version. This is the root source of truth.

    schema_name = "brighteyes_mcs_flim"
      # Required. Allows readers to distinguish this from generic HDF5 files.

    file_kind = "calibrated_measurement"
      # Required. This file is the output of calibrate_h5_file and contains
      # both data and calibration products.

    default = "/data/primary"
      # Required. Primary dataset path for simple readers.

    metadata = "/metadata"
      # Required. Path to normalized human-readable metadata.

    calibration = "/calibration"
      # Required. Path to calibration settings and outputs.

    comment = ""
      # Optional user comment copied from the acquisition or supplied during
      # calibration.

  /metadata
    # Normalized scalar metadata. This is the main place users should read
    # acquisition geometry, timing, channel counts, and provenance pointers.

  /axes
    # Reusable axis vectors: time axis, channel indices, and optional physical
    # coordinate axes. These are small, explicit, and remove ambiguity in array
    # dimensions.

  /data
    # Calibrated-file data products. Raw measured arrays are stored here with
    # clear names and axis attrs.

  /calibration
    # Calibration configuration, input summaries, and calibrated fit/delay/IRF
    # products.


/metadata
---------

/metadata.attrs:

  data_format_version = "0.0.5"
    # Mirror of /.attrs.data_format_version for convenience.

  source_data_format_version = "<input root data_format_version>"
    # Version of the uncalibrated input file, if known.

  acquisition_mode = "dfd" | "normal"
    # "dfd" means the time axis spans the laser period. "normal" means the
    # time axis spans the pixel dwell time.

  dfd_active = True | False
    # Boolean equivalent of acquisition_mode == "dfd".

  comment = ""
    # Human-readable acquisition or calibration comment.

  nx = <int>
  ny = <int>
  nz = <int>
    # Image dimensions. The raw data layout is
    # (repetition, z, y, x, time_bin, channel).

  nrep = <int>
    # Number of repetitions.

  timebins = <int>
    # Number of histogram/time bins per pixel. Must match
    # /data/primary.shape[-2].

  primary_channel_count = <int>
    # Detector channels in /data/primary. For a 5x5 SPAD array this is 25.

  extra_channel_count = <int>
    # Auxiliary channels in /data/extra, such as sync or laser-monitor channels.

  data_primary_path = "/data/primary"
    # Path to the main photon-count data.

  data_channels_extra_path = "/data/extra"
    # Path to auxiliary channels. Omit or set to "" if there are no extra
    # channels.

  data_axes = "repetition,z,y,x,time_bin,channel"
    # Axis order for /data/primary.

  extra_data_axes = "repetition,z,y,x,time_bin,extra_channel"
    # Axis order for /data/extra.

  range_x_um = <float>
  range_y_um = <float>
  range_z_um = <float>
    # Physical scan ranges in micrometers.

  pixel_size_x_um = range_x_um / nx
  pixel_size_y_um = range_y_um / ny
  pixel_size_z_um = range_z_um / nz
    # Physical pixel sizes in micrometers. If nz == 1 and range_z_um == 0,
    # pixel_size_z_um should be 0.0.

  laser_freq_mhz = <float>
    # Laser repetition frequency in MHz. Required for DFD mode. Recommended
    # for all FLIM files.

  laser_period_ns = 1000.0 / laser_freq_mhz
    # Laser period in ns. Required for DFD mode.

  time_resolution_us = <float>
    # Acquisition hardware time resolution in microseconds when available.
    # Preserve this separately from time_bin_in_ns because DFD files use the
    # laser period for the histogram time axis.

  pixel_dwell_time_us = <float>
    # Pixel dwell time in microseconds.
    # Existing MCS metadata often gives:
    #   pixel_dwell_time_us = time_resolution_us * timebins

  pixel_dwell_time_in_ns = pixel_dwell_time_us * 1000.0
    # Same dwell time in ns. Keep this spelling; do not use "dewell".

  time_bin_in_ns = <float>
    # Width of one FLIM histogram bin in ns.
    # Formula depends on acquisition mode:
    #   DFD mode:
    #     time_bin_in_ns = laser_period_ns / timebins
    #   Normal mode:
    #     time_bin_in_ns = pixel_dwell_time_us / timebins * 1000.0

  time_bin_source = "laser_period_ns / timebins" | "pixel_dwell_time_us / timebins * 1000"
    # Human-readable formula used to compute time_bin_in_ns.

  frame_time_s = pixel_dwell_time_us * nx * ny * nz / 1e6
    # Duration of one frame/volume in seconds.

  frame_rate_hz = 1.0 / frame_time_s
    # Frame rate in Hz. Store NaN if frame_time_s is zero or unavailable.

  acquisition_duration_s = frame_time_s * nrep
    # Approximate full acquisition duration in seconds.

  bitfile = "<path or name>"
    # FPGA bitfile path/name from acquisition metadata. Useful for inferring
    # DFD parameters when they are encoded in the bitfile name.

  spad_number_of_channels = <int>
    # Total configured SPAD detector channels.

  laser_enable_0 = True | False
  laser_enable_1 = True | False
  laser_enable_2 = True | False
  laser_enable_3 = True | False
    # Laser enable state copied from acquisition settings when present.

  scanner_calib_x = <float>
  scanner_calib_y = <float>
  scanner_calib_z = <float>
    # Scanner calibration factors from acquisition metadata.

  raw_configuration_gui_path = "/metadata/raw/configuration_gui"
  raw_configuration_gui_before_start_path = "/metadata/raw/configuration_gui_before_start"
  raw_configuration_fpga_path = "/metadata/raw/configuration_fpga"
  raw_configuration_spad_fcs_manager_path = "/metadata/raw/configuration_spad_fcs_manager"
    # Pointers to raw instrument metadata snapshots.

  metadata_warnings_json = "[]"
    # JSON list of normalization warnings. Example:
    # ["raw DFD bin count disagrees with data.shape[-2]; using data shape"]

/calibration/results/<product>
------------------------------

The writer should create one result group per calibrated data product.

Required products for the current workflow:
  /calibration/results/primary
    # Calibration results for /data/primary.

  /calibration/results/extra
    # Calibration results for /data/extra when auxiliary channels are
    # calibrated. Omit if no auxiliary data are present or calibrated.

The structure below applies to every <product>.

/calibration/results/<product>.attrs:
  product_name = "primary" | "extra"
  source_data_path = "/data/primary" | "/data/extra"
  source_key = "data" | "data_channels_extra"
  reference_key = "<source reference dataset key>"
  reference_type = "ref" | "irf"
  channel_count = <int>
  channel_count_calibrated = <int>
  channel_axis = -1
  histogram_axes = "time_bin,channel"
  timebins = /metadata.attrs.timebins
  time_axis_path = "/axes/time_ns"
  bin_width_in_ns = /metadata.attrs.time_bin_in_ns
  laser_freq_mhz = /metadata.attrs.laser_freq_mhz
  laser_period_ns = /metadata.attrs.laser_period_ns
  data_shape_json = "[nrep, nz, ny, nx, timebins, channel_count]"
  reference_shape_json = "[nrep, nz, ny, nx, timebins, channel_count]"
  channel_skew_fit_reference_product = "primary"
  channel_skew_fit_reference_channel_resolved = <int>

  channel_skew_fit_reference_data_key = "<source key>"

Required normalization rules
----------------------------

1. Determine array dimensions from datasets first.
   The writer should trust /data/primary.shape for nx, ny, nz, timebins, and
   channel_count when raw metadata disagrees.

2. Preserve raw metadata, even when inconsistent.
   Store raw GUI/FPGA attrs under /metadata/raw. Do not silently discard stale
   values; record conflicts in /metadata.attrs.metadata_warnings_json.

3. Use numeric NaN for missing numeric calibration inputs.
   Do not store the string "None" for numeric attrs such as initial_tau_ns or
   tau_ref_input_ns.

4. Use explicit units in names or attrs.
   Preferred names:
     laser_freq_mhz
     laser_period_ns
     time_bin_in_ns
     pixel_dwell_time_us
     pixel_dwell_time_in_ns
     pixel_size_x_um, pixel_size_y_um, pixel_size_z_um

5. Compute time_bin_in_ns from the correct physical clock.
   DFD mode:
     time_bin_in_ns = laser_period_ns / timebins
   Normal mode:
     time_bin_in_ns = pixel_dwell_time_us / timebins * 1000.0

6. Store structured attrs as JSON strings.
   HDF5 attrs are not equally portable for nested Python dict/list objects.
   Use *_json attr names for mappings, lists, p0, bounds, model kwargs, and
   warning lists.

7. Prefer UTF-8 string datasets over object dtype for text arrays.
   In particular, /calibration/results/<product>/fit/irf_type should be a
   variable-length UTF-8 string dataset.
