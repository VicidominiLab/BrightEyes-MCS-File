BrightEyes HDF5 schema v0.1.2
=============================

Status:
  Proposed common schema for new BrightEyes-MCS raw acquisitions and
  BrightEyes-MCS-File calibrated outputs.

Scope:
  This schema is acquisition-aware. It explicitly represents:

  - normal raster digital FIFO data;
  - DFD digital FIFO data;
  - normal circular scans, where circular motion is a spatial sub-scan and is
    not DFD sync data;
  - optional digital auxiliary data previously called data_channels_extra;
  - optional analog FIFO data previously called data_analog;
  - raw acquisition metadata and normalized human-readable metadata;
  - calibration metadata and results added by BrightEyes-MCS-File.

Main rule:
  BrightEyes-MCS writes raw files with data_format_version = "0.1.2",
  file_kind = "raw_acquisition", contains_calibration = False.

  BrightEyes-MCS-File calibrate_h5_file reads a raw acquisition and writes a
  calibrated file with data_format_version = "0.1.2",
  file_kind = "calibrated_measurement", contains_calibration = True. The
  original raw data values are preserved and calibration information is added
  under /calibration.

  A raw acquisition can be analog-only: /data/analog can exist without
  /data/primary. When /data/primary exists, /data/extra must also exist because
  BrightEyes-MCS digital data always include the auxiliary digital channels.


Terminology
-----------

Primary digital data:
  Main detector-channel counts from the BrightEyes-MCS digital FIFO.
  Canonical path: /data/primary.

Extra digital data:
  Auxiliary digital FIFO channels. In older files these are named
  data_channels_extra. Canonical path: /data/extra.

Analog data:
  FIFOAnalog samples. In older files these are named data_analog. Canonical
  path: /data/analog.

DFD mode:
  Digital Frequency Domain acquisition. The time axis spans the laser period.
  DFD is an acquisition mode, not a separate dataset name. Do not create a
  special data_dfd dataset.

Normal mode:
  Non-DFD acquisition. The time axis is derived from dwell time and the number
  of time bins.

Circular scan:
  Per-raster-pixel spatial sub-scan controlled by circular_active,
  circular_radius_nm, circular_points, circular_repetition, and the scan
  voltage arrays. Circular scan axes are explicit in the canonical data layout.


Repository responsibilities
---------------------------

BrightEyes-MCS:
  Writes the raw acquisition file.

  Required root identity:
    data_format_version = "0.1.2"
    schema_name = "brighteyes_mcs_h5"
    schema_variant = "acquisition_aware"
    file_kind = "raw_acquisition"
    contains_calibration = False
    raw_data_preserved = True

  Required behavior:
    - Write /metadata, /axes, /data, and optional /views.
    - Do not write /calibration.
    - Preserve raw GUI, FPGA, and SPAD/FCS manager settings under
      /metadata/raw.
    - Write normalized human-readable metadata under /metadata.attrs and
      /metadata/acquisition.
    - Use the canonical 6D data layout for non-circular acquisitions.
    - Use the canonical 8D data layout only when circular scanning is active.

BrightEyes-MCS-File:
  Reads raw acquisitions and writes calibrated measurements.

  Required root identity:
    data_format_version = "0.1.2"
    schema_name = "brighteyes_mcs_h5"
    schema_variant = "acquisition_aware"
    file_kind = "calibrated_measurement"
    contains_calibration = True
    raw_data_preserved = True

  Required behavior:
    - Preserve the raw data arrays without changing their numeric values.
    - Keep /data as the source data used for calibration.
    - Add /calibration with calibration settings, provenance, inputs, and
      results.
    - Keep normalized metadata and raw metadata snapshots.
    - Support old 0.0.1 files as input, but emit v0.1.2 by default.


Root layout
-----------

/
  .attrs:
    data_format_version = "0.1.2"
      # Required schema version.

    schema_name = "brighteyes_mcs_h5"
      # Stable schema identifier shared by BrightEyes-MCS and
      # BrightEyes-MCS-File.

    schema_variant = "acquisition_aware"
      # Required. Means circular scan axes, analog data, DFD timing, and raw
      # metadata are represented explicitly.

    file_kind = "raw_acquisition" | "calibrated_measurement"
      # raw_acquisition: produced directly by BrightEyes-MCS.
      # calibrated_measurement: produced by BrightEyes-MCS-File
      # calibrate_h5_file.

    contains_calibration = True | False
      # False for raw BrightEyes-MCS output. True for calibrated output.

    raw_data_preserved = True
      # Required. The original acquisition data values are preserved.

    default = "/data/primary"
      # Default dataset for simple HDF5 viewers when primary digital data
      # exists. If the file contains only analog data, use "/data/analog".

    metadata = "/metadata"
    axes = "/axes"
    data = "/data"
    calibration = "/calibration"
      # Major group pointers. calibration is present only when
      # contains_calibration = True.

    comment = ""
      # Free acquisition or calibration comment.

  groups:
    /metadata
    /axes
    /data
    /views
    /calibration


Required top-level groups
-------------------------

/metadata:
  Human-readable normalized metadata plus exact raw metadata snapshots.

/axes:
  Physical and logical axes used by /data and /calibration.

/data:
  Canonical raw data arrays. These arrays remain raw counts or raw ADC values
  even in a calibrated file.

/views:
  Optional compatibility views, preferably HDF5 virtual datasets.

/calibration:
  Present only for file_kind = "calibrated_measurement".


Canonical data layouts
----------------------

Non-circular acquisitions use this 6D axis order:

  (repetition, z, y, x, time_bin, channel)

Circular acquisitions use this 8D axis order:

  (repetition, z, y, x, circular_repetition, circular_point, time_bin, channel)

If circular scan is inactive:

  subpixel_scan_mode = "none"
  data_layout = "raster_6d"
  circular axes are not present in /data shapes

If circular scan is active:

  circular_repetition_count = configured circular repetitions
  circular_point_count = configured circular points
  subpixel_scan_mode = "circular"
  data_layout = "circular_8d"

This removes the ambiguity of the older flattened layout where
circular_repetition * circular_point * time_bin were merged into one time-like
axis. The circular axes are added only when they are physically meaningful.


/metadata.attrs
---------------

Core identity:

  data_format_version = "0.1.2"
  schema_name = "brighteyes_mcs_h5"
  schema_variant = "acquisition_aware"
  file_kind = "raw_acquisition" | "calibrated_measurement"
  contains_calibration = True | False
  raw_data_preserved = True
  comment = ""

Data product presence:

  digital_fifo_active = True | False
  analog_fifo_active = True | False
  data_layout = "raster_6d" | "circular_8d"

  primary_present = True | False
  extra_present = True | False
  analog_present = True | False

Rules:
  - analog_present can be True while primary_present is False.
  - primary_present True implies extra_present True.
  - extra_present without primary_present is invalid for BrightEyes-MCS output.

Acquisition mode:

  acquisition_mode = "normal" | "dfd"
  dfd_active = True | False
  timing_reference = "pixel_dwell" | "laser_period"
  dfd_laser_sync_debug = True | False
  dfd_trigger_selector = <int or -1>

Spatial raster dimensions:

  nrep = <int>
  nz = <int>
  ny = <int>
  nx = <int>

  range_x_um = <float>
  range_y_um = <float>
  range_z_um = <float>

  offset_x_um = <float>
  offset_y_um = <float>
  offset_z_um = <float>

  pixel_size_x_um = range_x_um / nx
  pixel_size_y_um = range_y_um / ny
  pixel_size_z_um = range_z_um / nz

  snake_walk_xy = True | False
  snake_walk_z = True | False

Circular scan:

  subpixel_scan_mode = "none" | "circular"
  circular_active = True | False
  circular_radius_nm = <float or 0.0>
  circular_radius_um = circular_radius_nm / 1000.0
  circular_point_count = <int or 0>
  circular_repetition_count = <int or 0>
  circular_samples_per_pixel = <int>
    # 1 for non-circular acquisitions.
    # circular_point_count * circular_repetition_count for circular scans.

  circular_phase_axis_path = "/axes/circular_phase_rad"
  circular_x_um_axis_path = "/axes/circular_x_um"
  circular_y_um_axis_path = "/axes/circular_y_um"
  circular_z_um_axis_path = "/axes/circular_z_um"
  scan_x_voltages_path = "/axes/scan_x_voltages"
  scan_y_voltages_path = "/axes/scan_y_voltages"
  scan_z_voltages_path = "/axes/scan_z_voltages"

Timing:

  base_timebins_per_point = <int>
    # Time bins configured for one raster or circular sub-point before any DFD
    # digital clk_multiplier reduction.

  digital_timebins_per_point = <int>
    # Time bins stored in /data/primary and /data/extra for one circular point.

  analog_timebins_per_point = <int>
    # Time bins stored in /data/analog for one circular point.

  timebins = digital_timebins_per_point
    # Convenience alias for primary digital data. If the file is analog-only,
    # this equals analog_timebins_per_point.

  time_resolution_us = <float>
    # Hardware time resolution in microseconds per bin in normal mode.

  laser_freq_mhz = <float or NaN>
  laser_period_ns = 1000.0 / laser_freq_mhz

  digital_clk_multiplier = <int>
    # Current BrightEyes-MCS applies this to digital data in DFD mode.

  digital_clk_multiplier_scope = "time_bin" | "flattened_sample"
    # New writers should use "time_bin". When migrating old files, record
    # "flattened_sample" if the source flattened circular samples before the
    # clk_multiplier reduction.

  dwell_time_per_circular_point_us = <float>
    # Normal mode:
    #   digital_time_bin_in_ns / 1000 * digital_timebins_per_point
    # or:
    #   base_timebins_per_point * time_resolution_us

  pixel_dwell_time_us = <float>
    # Total dwell time for one raster pixel including all circular points and
    # repetitions:
    #   dwell_time_per_circular_point_us * circular_samples_per_pixel

  pixel_dwell_time_in_ns = pixel_dwell_time_us * 1000.0

  digital_time_bin_in_ns = <float>
    # DFD mode:
    #   laser_period_ns / digital_timebins_per_point
    # Normal mode, no circular scan:
    #   pixel_dwell_time_us / digital_timebins_per_point * 1000.0
    # Normal mode, circular scan:
    #   pixel_dwell_time_us /
    #   (circular_samples_per_pixel * digital_timebins_per_point) * 1000.0
    # Equivalent normal-mode expression:
    #   dwell_time_per_circular_point_us / digital_timebins_per_point * 1000.0

  analog_time_bin_in_ns = <float>
    # DFD mode:
    #   laser_period_ns / analog_timebins_per_point
    # Normal mode:
    #   dwell_time_per_circular_point_us / analog_timebins_per_point * 1000.0

  line_time_s = pixel_dwell_time_us * nx / 1e6
  frame_time_s = pixel_dwell_time_us * nx * ny / 1e6
  volume_time_s = frame_time_s * nz
  acquisition_duration_s = volume_time_s * nrep

Channels:

  primary_channel_count = <int>
  detector_grid_shape_json = "[5, 5]" | "[7, 7]" | "[]"

  extra_channel_count = <int>
  extra_channel_labels_json = "[]"

  analog_channel_count = <int>
  analog_channel_labels_json = "[\"analog_A\", \"analog_B\"]"

Analog configuration summary:

  analog_input_a_selector = <int or -1>
  analog_input_b_selector = <int or -1>
  analog_a_differential = True | False
  analog_b_differential = True | False
  analog_a_integrate_json = "[...]"
  analog_a_invert_json = "[...]"
  analog_b_integrate_json = "[...]"
  analog_b_invert_json = "[...]"

Provenance:

  created_by_package = "brighteyes_mcs" | "brighteyes_mcs_file"
  created_by_version = "<package version or unknown>"
  created_at_iso8601 = "<timestamp>"
  source_data_file = "<path or empty>"
  source_reference_file = "<path or empty>"
  output_file = "<path or empty>"
  source_data_format_version = "<version or empty>"
  metadata_warnings_json = "[]"

Raw metadata pointers:

  raw_configuration_gui_path = "/metadata/raw/configuration_gui"
  raw_configuration_gui_before_start_path = "/metadata/raw/configuration_gui_before_start"
  raw_configuration_fpga_path = "/metadata/raw/configuration_fpga"
  raw_configuration_spad_fcs_manager_path = "/metadata/raw/configuration_spad_fcs_manager"

/axes
-----

Raster axes:

/axes/repetition_index:
  shape = (nrep,)
  dtype = int64

/axes/z_um:
  shape = (nz,)
  dtype = float64
  attrs: units = "um", axis = "z"

/axes/y_um:
  shape = (ny,)
  dtype = float64
  attrs: units = "um", axis = "y"

/axes/x_um:
  shape = (nx,)
  dtype = float64
  attrs: units = "um", axis = "x"

Circular axes:
  Present only when circular_active = True. Omit these datasets for
  non-circular raster acquisitions.

/axes/circular_repetition_index:
  shape = (circular_repetition_count,)
  dtype = int64

/axes/circular_point_index:
  shape = (circular_point_count,)
  dtype = int64

/axes/circular_phase_rad:
  shape = (circular_point_count,)
  dtype = float64
  values = linspace(0, 2*pi, circular_point_count, endpoint=False)
  attrs: units = "rad", axis = "circular_point"

/axes/circular_x_um:
  shape = (circular_point_count,)
  dtype = float64
  values = cos(circular_phase_rad) * circular_radius_um
  attrs:
    units = "um"
    axis = "circular_point"
    long_name = "circular sub-scan x offset"

/axes/circular_y_um:
  shape = (circular_point_count,)
  dtype = float64
  values = sin(circular_phase_rad) * circular_radius_um
  attrs:
    units = "um"
    axis = "circular_point"
    long_name = "circular sub-scan y offset"

/axes/circular_z_um:
  shape = (circular_point_count,)
  dtype = float64
  values = zeros(circular_point_count)
  attrs:
    units = "um"
    axis = "circular_point"
    long_name = "circular sub-scan z offset"

/axes/scan_x_voltages:
  shape = (>= circular_point_count,)
  dtype = float64
  attrs:
    units = "V"
    source_register = "ScanXVoltages"
    note = "May include padded FPGA register values."

/axes/scan_y_voltages:
  shape = (>= circular_point_count,)
  dtype = float64
  attrs:
    units = "V"
    source_register = "ScanYVoltages"

/axes/scan_z_voltages:
  shape = (>= circular_point_count,)
  dtype = float64
  attrs:
    units = "V"
    source_register = "ScanZVoltages"

Time axes:

/axes/digital_time_ns:
  shape = (digital_timebins_per_point,)
  dtype = float64
  values = arange(digital_timebins_per_point) * digital_time_bin_in_ns
  attrs:
    units = "ns"
    axis = "time_bin"
    acquisition_mode = "normal" | "dfd"
    timing_reference = "pixel_dwell" | "laser_period"
    clk_multiplier = digital_clk_multiplier

/axes/analog_time_ns:
  shape = (analog_timebins_per_point,)
  dtype = float64
  values = arange(analog_timebins_per_point) * analog_time_bin_in_ns
  attrs:
    units = "ns"
    axis = "time_bin"
    acquisition_mode = "normal" | "dfd"
    timing_reference = "pixel_dwell" | "laser_period"

Channel axes:

  shape = (primary_channel_count,)
  dtype = int64

/axes/extra_channel_index:
  shape = (extra_channel_count,)
  dtype = int64

/axes/analog_channel_index:

/data

/data.attrs:
  layout = "raster_6d" | "circular_8d"
  axes = "repetition,z,y,x,time_bin,channel"
       | "repetition,z,y,x,circular_repetition,circular_point,time_bin,channel"
  primary_present = True | False
  extra_present = True | False
  analog_present = True | False
  thumbnail_present = True | False

/data/primary:
  required_if = digital_fifo_active
  absent_if = analog-only acquisition
  non_circular_shape = (
    nrep,
    nz,
    ny,
    nx,
    digital_timebins_per_point,
    primary_channel_count
  )
  circular_shape = (
    nrep,
    nz,
    ny,
    nx,
    circular_repetition_count,
    circular_point_count,
    digital_timebins_per_point,
    primary_channel_count
  )
  dtype = uint16 or source-compatible integer type
  attrs:
    long_name = "primary digital detector counts"
    source_fifo = "FIFO"
    source_legacy_key = "data"
    acquisition_mode = "normal" | "dfd"
    subpixel_scan_mode = "none" | "circular"
    axes = "repetition,z,y,x,time_bin,detector_channel"
         | "repetition,z,y,x,circular_repetition,circular_point,time_bin,detector_channel"
    units = "counts"
    time_axis_path = "/axes/digital_time_ns"
    detector_channel_axis_path = "/axes/detector_channel_index"
    circular_point_axis_path = "/axes/circular_point_index" | ""
    circular_repetition_axis_path = "/axes/circular_repetition_index" | ""
    calibration_result_path = "/calibration/results/primary" | ""

/data/extra:
  required_if = /data/primary is present
    # In BrightEyes-MCS, primary digital data always come with extra digital
    # channels. A file with /data/primary but no /data/extra is incomplete.
  absent_if = analog-only acquisition
  non_circular_shape = (
    nrep,
    nz,
    ny,
    nx,
    digital_timebins_per_point,
    extra_channel_count
  )
  circular_shape = (
    nrep,
    nz,
    ny,
    nx,
    circular_repetition_count,
    circular_point_count,
    digital_timebins_per_point,
    extra_channel_count
  )
  dtype = uint8 or source-compatible integer type
  attrs:
    long_name = "auxiliary digital FIFO counts"
    source_fifo = "FIFO"
    source_legacy_key = "data_channels_extra"
    acquisition_mode = "normal" | "dfd"
    subpixel_scan_mode = "none" | "circular"
    axes = "repetition,z,y,x,time_bin,extra_channel"
         | "repetition,z,y,x,circular_repetition,circular_point,time_bin,extra_channel"
    units = "counts"
    time_axis_path = "/axes/digital_time_ns"
    extra_channel_axis_path = "/axes/extra_channel_index"
    channel_labels_json = "[]"
    calibration_result_path = "/calibration/results/extra" | ""

/data/analog:
  required_if = analog_fifo_active
  can_exist_without = "/data/primary"
    # Analog-only raw acquisitions are valid.
  non_circular_shape = (
    nrep,
    nz,
    ny,
    nx,
    analog_timebins_per_point,
    analog_channel_count
  )
  circular_shape = (
    nrep,
    nz,
    ny,
    nx,
    circular_repetition_count,
    circular_point_count,
    analog_timebins_per_point,
    analog_channel_count
  )
  dtype = int32 or source-compatible numeric type
  attrs:
    long_name = "analog FIFO data"
    source_fifo = "FIFOAnalog"
    source_legacy_key = "data_analog"
    acquisition_mode = "normal" | "dfd"
    subpixel_scan_mode = "none" | "circular"
    axes = "repetition,z,y,x,time_bin,analog_channel"
         | "repetition,z,y,x,circular_repetition,circular_point,time_bin,analog_channel"
    units = "adc_counts"
    time_axis_path = "/axes/analog_time_ns"
    analog_channel_axis_path = "/axes/analog_channel_index"
    channel_labels_json = "[\"analog_A\", \"analog_B\"]"
    calibration_result_path = ""

/data/thumbnail:
  optional = True
  attrs:
    long_name = "preview thumbnail"


/views
------

Optional compatibility views:

/views/flat_sample/primary:
  shape = (nrep, nz, ny, nx, digital_flattened_bins_per_pixel, primary_channel_count)
  attrs:
    source = "/data/primary"
    flattened_axes = "circular_repetition,circular_point,time_bin"
    use = "compatibility view for code expecting the older 6D layout"

/views/flat_sample/extra:
  shape = (nrep, nz, ny, nx, digital_flattened_bins_per_pixel, extra_channel_count)
  attrs:
    source = "/data/extra"
    flattened_axes = "circular_repetition,circular_point,time_bin"

/views/flat_sample/analog:
  shape = (nrep, nz, ny, nx, analog_flattened_bins_per_pixel, analog_channel_count)
  attrs:
    source = "/data/analog"
    flattened_axes = "circular_repetition,circular_point,time_bin"

Recommendation:
  Use HDF5 virtual datasets when practical. If virtual datasets are too complex
  initially, omit /views and let readers reshape from /data explicitly.


/calibration
------------

Only present when file_kind = "calibrated_measurement".

/calibration.attrs:
  calibrated_products_json = "[\"primary\", \"extra\"]"
  reference_type = "ref" | "irf"

Mode examples
-------------

1. Raw normal raster digital-only file from BrightEyes-MCS:

  / attrs:
    file_kind = "raw_acquisition"
    contains_calibration = False

  /metadata.attrs:
    acquisition_mode = "normal"
    subpixel_scan_mode = "none"
    data_layout = "raster_6d"
    digital_fifo_active = True
    analog_fifo_active = False
    primary_present = True
    extra_present = True

  /data/primary.shape =
    (nrep, nz, ny, nx, digital_timebins_per_point, primary_channel_count)

  /data/extra.shape =
    (nrep, nz, ny, nx, digital_timebins_per_point, extra_channel_count)

2. Raw DFD raster digital-only file from BrightEyes-MCS:

  /metadata.attrs:
    acquisition_mode = "dfd"
    data_layout = "raster_6d"
    timing_reference = "laser_period"
    laser_freq_mhz = 40.0
    laser_period_ns = 25.0
    digital_time_bin_in_ns = laser_period_ns / digital_timebins_per_point

  /data/primary remains /data/primary. Do not rename it to data_dfd.

3. Raw normal circular scan from BrightEyes-MCS:

  /metadata.attrs:
    acquisition_mode = "normal"
    subpixel_scan_mode = "circular"
    data_layout = "circular_8d"
    circular_active = True
    circular_point_count = N
    circular_repetition_count = R

  /data/primary.shape =
    (nrep, nz, ny, nx, R, N, digital_timebins_per_point, primary_channel_count)

  This is a normal circular scan, not sync DFD data.

4. Raw analog-only raster file from BrightEyes-MCS:

  /metadata.attrs:
    analog_fifo_active = True
    analog_present = True
    primary_present = False
    extra_present = False
    data_layout = "raster_6d"

  /data/analog.shape =
    (nrep, nz, ny, nx, analog_timebins_per_point, analog_channel_count)

5. Raw analog-only circular file from BrightEyes-MCS:

  /metadata.attrs:
    analog_fifo_active = True
    analog_present = True
    primary_present = False
    extra_present = False
    subpixel_scan_mode = "circular"
    data_layout = "circular_8d"

  /data/analog.shape =
    (nrep, nz, ny, nx, R, N, analog_timebins_per_point, analog_channel_count)

6. Calibrated file from BrightEyes-MCS-File:

  / attrs:
    file_kind = "calibrated_measurement"
    contains_calibration = True
    raw_data_preserved = True

  /data:
    Same raw acquisition data values used for calibration.

  /calibration:
    Calibration settings, provenance, inputs, and results.


Compatibility policy
--------------------

Readers:
  Readers should support:
    - v0.1.2 canonical paths;
    - older files with top-level data, data_channels_extra, data_analog,
      configurationGUI, configurationFPGA, configurationGUI_beforeStart, and
      configurationSpadFCSmanager.

Writers:
  New BrightEyes-MCS raw files should write v0.1.2 directly.
  New calibrate_h5_file outputs should write v0.1.2 by default.

Old input conversion:
  If an old input has shape:

    (nrep, nz, ny, nx, flat_sample, channel)

  and:

    flat_sample == circular_repetition_count
                   * circular_point_count
                   * timebins_per_point

  then the v0.1.2 canonical shape should be:

    (nrep, nz, ny, nx,
     circular_repetition_count,
     circular_point_count,
     timebins_per_point,
     channel)

  If the product cannot be safely split, keep enough metadata to warn the user
  and expose a flat_sample compatibility view.

Metadata conflicts:
  When metadata fields disagree, do not silently choose one. Normalize the best
  value for analysis, preserve all raw values under /metadata/raw, and append a
  human-readable warning to metadata_warnings_json.


Implementation notes
--------------------

Use JSON strings for structured HDF5 attributes:
  detector_grid_shape_json, *_labels_json, warnings, and product maps should
  be valid JSON strings.

Use normal HDF5 numeric datasets for axes and calibration arrays.

Use variable-length UTF-8 string datasets for string arrays such as irf_type.

Avoid object dtype arrays in new files.

Do not use a dataset name to encode acquisition mode:
  Use /metadata.attrs.acquisition_mode and timing_reference instead.

Do not hide circular scanning inside the time axis:
  Keep circular_repetition and circular_point as explicit axes only when
  circular scanning is active. Use simple 6D datasets otherwise.

Do not treat normal circular scan as DFD sync data:
  The mode is acquisition_mode = "normal" and subpixel_scan_mode = "circular".
