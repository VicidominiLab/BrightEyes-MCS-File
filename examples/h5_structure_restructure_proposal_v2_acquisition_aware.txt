BrightEyes calibrated HDF5 schema v0.0.5 - acquisition-aware proposal
=====================================================================

Status:
  New proposal after checking BrightEyes-MCS acquisition code.

Purpose:
  Define a clean v0.0.5 calibrated HDF5 output format for calibrate_h5_file
  that can represent:

  - regular raster digital FIFO data;
  - DFD digital FIFO data;
  - normal circular scan data, where circular points/repetitions are not DFD
    sync data and should not be confused with DFD;
  - optional data_channels_extra from the digital FIFO;
  - optional data_analog from FIFOAnalog;
  - circular scanning metadata, including radius, circular points,
    circular repetitions, and the generated scan voltage arrays;
  - raw BrightEyes-MCS GUI/FPGA/SPAD metadata for traceability.

Main design change compared with the previous proposal:
  The canonical data arrays are 8D instead of 6D:

    (repetition, z, y, x, circular_repetition, circular_point, time_bin, channel)

  If circular scanning is inactive:

    circular_repetition = 1
    circular_point = 1

  This avoids the current ambiguity where BrightEyes-MCS flattens
  circular_repetition * circular_point * time_bin into a single time-like axis.


Terminology
-----------

Digital FIFO:
  BrightEyes-MCS FIFO named "FIFO". It can produce:
    - primary detector channels, currently written as source key "data";
    - auxiliary digital channels, currently written as source key
      "data_channels_extra".

Analog FIFO:
  BrightEyes-MCS FIFO named "FIFOAnalog". It can produce:
    - analog channels, currently written as source key "data_analog".

DFD mode:
  Digital Frequency Domain acquisition. DFD changes the time axis meaning:
  the per-point histogram spans the laser period. DFD is an acquisition mode,
  not a separate required dataset path.

Normal mode:
  Non-DFD acquisition. The per-point time axis comes from the hardware time
  resolution and pixel dwell settings.

Circular sub-scan:
  A per-pixel circular scan generated by BrightEyes-MCS using:
    circular_active / CircularMotionActivate
    circular_radius_nm
    circular_points / #circular_points
    circular_repetition / #circular_rep
    ScanXVoltages, ScanYVoltages, ScanZVoltages

  This is not DFD sync data. It is a spatial sub-sampling mode inside each
  raster pixel.


Root
----

/
  .attrs:
    data_format_version = "0.0.5"
      # Required root schema version.

    schema_name = "brighteyes_mcs_flim"
      # Required schema identifier.

    schema_variant = "acquisition_aware_clean"
      # Required. Tells readers this file uses explicit circular and analog
      # axes.

    file_kind = "calibrated_measurement"
      # Required. calibrate_h5_file output containing data and calibration.

    default = "/data/primary"
      # Required primary data product for simple readers.

    metadata = "/metadata"
    axes = "/axes"
    data = "/data"
    calibration = "/calibration"
      # Required pointers to major groups.

    comment = ""
      # Optional acquisition or calibration comment.


Top-level groups
----------------

/metadata
  # Human-readable normalized metadata and raw metadata snapshots.

/axes
  # Physical and logical axis datasets used by /data and /calibration.

/data
  # Canonical data products. Digital, auxiliary digital, and analog data live
  # here.

/views
  # Optional reader-friendly views. These can be HDF5 virtual datasets when
  # possible. They are not the canonical storage.

/calibration
  # Calibration settings, input summaries, and per-product calibration results.


/metadata.attrs
---------------

Core file identity:

  data_format_version = "0.0.5"
  schema_variant = "acquisition_aware_clean"
  source_data_format_version = "<input version or unknown>"
  comment = ""

Acquisition mode:

  acquisition_mode = "normal" | "dfd"
    # "normal": time bins represent hardware time-resolution bins within the
    # dwell time.
    # "dfd": time bins represent laser-period phase bins.

  dfd_active = True | False
    # Boolean equivalent of acquisition_mode == "dfd".

  timing_reference = "pixel_dwell" | "laser_period"
    # "pixel_dwell" for normal mode.
    # "laser_period" for DFD mode.

  dfd_laser_sync_debug = True | False
    # From DFD_LaserSyncDebug when present. This describes debug/sync handling;
    # it should not rename the primary data into data_dfd.

  dfd_trigger_selector = <int or -1>
    # From DFD_Trig_Selector when present.

Data-product presence:

  digital_fifo_active = True | False
    # True when FIFO data are present.

  analog_fifo_active = True | False
    # True when FIFOAnalog/data_analog are present.

  data_products_json = "[\"primary\", \"extra\", \"analog\"]"
    # JSON list of products present in /data. Omit names that are absent.

  data_primary_path = "/data/primary"
  data_channels_extra_path = "/data/extra"
  data_analog_path = "/data/analog"
    # Set absent optional paths to "".

Spatial raster dimensions:

  nx = <int>
  ny = <int>
  nz = <int>
  nrep = <int>
    # Raster dimensions and repetitions.

  range_x_um = <float>
  range_y_um = <float>
  range_z_um = <float>

  offset_x_um = <float>
  offset_y_um = <float>
  offset_z_um = <float>

  pixel_size_x_um = range_x_um / nx
  pixel_size_y_um = range_y_um / ny
  pixel_size_z_um = range_z_um / nz

  snake_walk_xy = True | False
  snake_walk_z = True | False
    # From snake and snake_z. Needed to decode or verify acquisition order.

Circular sub-scan:

  subpixel_scan_mode = "none" | "circular"
    # "circular" when circular_active or CircularMotionActivate is true.

  circular_active = True | False

  circular_radius_nm = <float>
  circular_radius_um = circular_radius_nm / 1000.0

  circular_point_count = <int>
    # Number of points around the circle. From circular_points or
    # #circular_points.

  circular_repetition_count = <int>
    # Number of repetitions of the circular sub-scan per raster pixel. From
    # circular_repetition or #circular_rep.

  circular_samples_per_pixel = circular_point_count * circular_repetition_count

  circular_axes = "circular_repetition,circular_point"
    # Canonical order of the explicit circular axes in /data arrays.

  circular_phase_axis_path = "/axes/circular_phase_rad"
  circular_x_um_axis_path = "/axes/circular_x_um"
  circular_y_um_axis_path = "/axes/circular_y_um"
  circular_z_um_axis_path = "/axes/circular_z_um"
  scan_x_voltages_path = "/axes/scan_x_voltages"
  scan_y_voltages_path = "/axes/scan_y_voltages"
  scan_z_voltages_path = "/axes/scan_z_voltages"

Timing:

  base_timebins_per_point = <int>
    # GUI/register #timebinsPerPixel before multiplication by circular points
    # and repetitions.

  digital_timebins_per_point = <int>
    # Saved time bins per circular point for digital FIFO data.
    # In current BrightEyes-MCS this may be:
    #   base_timebins_per_point // digital_clk_multiplier

  analog_timebins_per_point = <int>
    # Saved time bins per circular point for analog FIFO data.
    # In current BrightEyes-MCS this is not divided by clk_multiplier.

  flattened_sample_bins_per_pixel = circular_samples_per_pixel * base_timebins_per_point
    # Documents the current acquisition-side flattened pointer space.

  digital_flattened_bins_per_pixel = circular_samples_per_pixel * digital_timebins_per_point
  analog_flattened_bins_per_pixel = circular_samples_per_pixel * analog_timebins_per_point
    # Useful for optional /views/flat_sample datasets.

  time_resolution_us = <float>
    # From GUI time_resolution or register Cx / clock_base.

  clock_base_mhz = <float>
    # Usually 40 MHz in current BrightEyes-MCS.

  clock_duration_cycles = <int>
    # From ClockDur when present.

  digital_clk_multiplier = <int>
    # Current BrightEyes-MCS applies this only to digital data, and only when
    # DFD is active.

  digital_clk_multiplier_scope = "time_bin" | "flattened_sample"
    # Target v0.0.5 writers should use "time_bin": decimation applies inside
    # each circular point. If converting a file produced by current
    # BrightEyes-MCS, record "flattened_sample" if the source applied the
    # multiplier after circular_repetition * circular_point * time_bin were
    # flattened together.

  laser_freq_mhz = <float or NaN>
  laser_period_ns = 1000.0 / laser_freq_mhz
    # Required for DFD analysis.

  digital_time_bin_in_ns = <float>
    # DFD:
    #   laser_period_ns / digital_timebins_per_point
    # Normal:
    #   time_resolution_us * 1000.0

  analog_time_bin_in_ns = <float>
    # DFD:
    #   laser_period_ns / analog_timebins_per_point
    # Normal:
    #   time_resolution_us * 1000.0

  dwell_time_per_circular_point_us = base_timebins_per_point * time_resolution_us
    # Physical dwell at one circular sub-point.

  pixel_dwell_time_us = dwell_time_per_circular_point_us * circular_samples_per_pixel
    # Total dwell per raster pixel including circular points and circular
    # repetitions.

  pixel_dwell_time_in_ns = pixel_dwell_time_us * 1000.0

  line_time_s = pixel_dwell_time_us * nx / 1e6
  frame_time_s = pixel_dwell_time_us * nx * ny / 1e6
  volume_time_s = frame_time_s * nz
  acquisition_duration_s = volume_time_s * nrep

Channels:

  primary_channel_count = <int>
    # Detector channels in /data/primary. Current standard values are 25 or 49.

  detector_grid_shape_json = "[5, 5]" | "[7, 7]"
    # Derived from primary_channel_count when it is a square detector.

  extra_channel_count = <int>
    # Digital auxiliary channels in /data/extra. Current BrightEyes-MCS writes
    # 2 extra digital channels.

  analog_channel_count = <int>
    # Analog channels in /data/analog. Current BrightEyes-MCS writes 2.

  extra_channel_labels_json = "[]"
    # Optional. Use labels such as ["aux_0", "laser_sync"] only when known.

  analog_channel_labels_json = "[\"analog_A\", \"analog_B\"]"
    # Recommended default labels for the two analog channels.

Analog configuration summary:

  analog_input_a_selector = <int or -1>
  analog_input_b_selector = <int or -1>
  analog_a_differential = True | False
  analog_b_differential = True | False
  analog_a_integrate_json = "[false, false, false, false, false, false, false, false]"
  analog_a_invert_json = "[false, false, false, false, false, false, false, false]"
    # Summarize the AnalogA/B settings copied in full under /metadata/raw.

Provenance and raw metadata pointers:

  source_data_file = "<path>"
  source_reference_file = "<path>"
  output_file = "<path>"
  created_by_package = "brighteyes_mcs_file"
  created_by_version = "<version>"
  created_at_iso8601 = "<timestamp>"

  raw_configuration_gui_path = "/metadata/raw/configuration_gui"
  raw_configuration_gui_before_start_path = "/metadata/raw/configuration_gui_before_start"
  raw_configuration_fpga_path = "/metadata/raw/configuration_fpga"
  raw_configuration_spad_fcs_manager_path = "/metadata/raw/configuration_spad_fcs_manager"

  metadata_warnings_json = "[]"
    # JSON list. Record conflicts such as raw DFD bin count disagreeing with
    # data shape.

  shape = (circular_point_count,)
  dtype = float64
  values = linspace(0, 2*pi, circular_point_count, endpoint=False)
  attrs:
    units = "rad"
    axis = "circular_point"

  shape = (circular_point_count,)
  dtype = float64

  shape = (circular_point_count,)
  dtype = float64

    units = "um"

/views
------

Optional views can make current algorithms easier to port while keeping
canonical data explicit.

/views/flat_sample/primary:
  optional = True
  shape = (nrep, nz, ny, nx, digital_flattened_bins_per_pixel, primary_channel_count)
  attrs:
    source = "/data/primary"
    flattened_axes = "circular_repetition,circular_point,time_bin"
    use = "compatibility view for algorithms expecting the previous 6D layout"

/views/flat_sample/extra:
  optional = True
  shape = (nrep, nz, ny, nx, digital_flattened_bins_per_pixel, extra_channel_count)
  attrs:
    source = "/data/extra"
    flattened_axes = "circular_repetition,circular_point,time_bin"

/views/flat_sample/analog:
  optional = True
  shape = (nrep, nz, ny, nx, analog_flattened_bins_per_pixel, analog_channel_count)
  attrs:
    source = "/data/analog"
    flattened_axes = "circular_repetition,circular_point,time_bin"

Recommendation:
  Use HDF5 virtual datasets for /views/flat_sample when possible. If virtual
  datasets are too complex initially, omit /views and let readers reshape
  explicitly from canonical /data arrays.


/calibration
------------

/calibration.attrs:
  calibrated_products_json = "[\"primary\", \"extra\"]"
    # Analog is not included unless a specific analog calibration is performed.

  reference_type = "ref" | "irf"
  reference_keys_json = "{\"primary\": \"data\", \"extra\": \"data_channels_extra\"}"
  fit_mode = "model_shift"
  fit_type = "circular" | "likelihood" | "curve_fit" | "curve_fit_circular"
  residual_error_metric = "rmse_normalized_histograms"
  C_ref = <float>
  tau_ref_input_ns = <float or NaN>
  irf_iterations = <int>
  eps = <float>
  regularization = <float>
  clean_irf = True | False
  irf_corrections_type = "median" | "single_ch" | "<other>"

  source_reduction_axes = "repetition,z,y,x,circular_repetition,circular_point"
    # Calibration histograms are normally produced by summing over all spatial
    # and circular axes, leaving (time_bin, channel).

  metadata_path = "/metadata"
  axes_path = "/axes"
  results_path = "/calibration/results"


/calibration/results/<product>
------------------------------

Valid FLIM products:
  primary
    # Results for /data/primary.

  extra
    # Optional results for /data/extra. Some auxiliary channels may be sync or
    # marker channels and may not be meaningful for lifetime calibration.

Result group attrs:

/calibration/results/<product>.attrs:
  product_name = "primary" | "extra"
  source_data_path = "/data/primary" | "/data/extra"
  source_key = "data" | "data_channels_extra"
  source_fifo = "FIFO"
  source_axes = "repetition,z,y,x,circular_repetition,circular_point,time_bin,channel"
  source_reduction_axes = "repetition,z,y,x,circular_repetition,circular_point"
  histogram_axes = "time_bin,channel"
  time_axis_path = "/axes/digital_time_ns"
  timebins = digital_timebins_per_point
  channel_count = <int>
  reference_key = "<source reference key>"
  reference_type = "ref" | "irf"
  bin_width_in_ns = /metadata.attrs.digital_time_bin_in_ns
  acquisition_mode = /metadata.attrs.acquisition_mode
  subpixel_scan_mode = /metadata.attrs.subpixel_scan_mode

Core datasets:

/calibration/results/<product>/channels/index:

Mode examples
-------------

1. Normal raster digital-only acquisition:

  acquisition_mode = "normal"
  subpixel_scan_mode = "none"
  circular_point_count = 1
  circular_repetition_count = 1
  digital_fifo_active = True
  analog_fifo_active = False
  /data/primary.shape =
    (nrep, nz, ny, nx, 1, 1, base_timebins_per_point, primary_channel_count)

2. Normal circular scan, not DFD sync:

  acquisition_mode = "normal"
  subpixel_scan_mode = "circular"
  circular_point_count > 1
  circular_repetition_count >= 1
  timing_reference = "pixel_dwell"
  digital_time_bin_in_ns = time_resolution_us * 1000.0
  /data/primary.shape =
    (nrep, nz, ny, nx, circular_repetition_count,
     circular_point_count, base_timebins_per_point, primary_channel_count)

  This is the case that would otherwise look "weird" in the current flat
  layout, because circular samples are folded into the time axis. In v0.0.5
  they are explicit axes.

3. DFD raster digital acquisition:

  acquisition_mode = "dfd"
  subpixel_scan_mode = "none"
  timing_reference = "laser_period"
  circular_point_count = 1
  circular_repetition_count = 1
  digital_time_bin_in_ns = laser_period_ns / digital_timebins_per_point
  /data/primary.shape =
    (nrep, nz, ny, nx, 1, 1, digital_timebins_per_point, primary_channel_count)

4. DFD circular digital plus analog acquisition:

  acquisition_mode = "dfd"
  subpixel_scan_mode = "circular"
  digital_fifo_active = True
  analog_fifo_active = True
  digital_clk_multiplier may be > 1
  digital_timebins_per_point may differ from analog_timebins_per_point

  /data/primary.shape =
    (nrep, nz, ny, nx, circular_repetition_count,
     circular_point_count, digital_timebins_per_point, primary_channel_count)

  /data/analog.shape =
    (nrep, nz, ny, nx, circular_repetition_count,
     circular_point_count, analog_timebins_per_point, analog_channel_count)


Writer rules
------------

1. Do not encode circular scan as hidden time bins in the canonical data.
   Split circular_repetition, circular_point, and time_bin axes explicitly.

2. New v0.0.5 writers should write canonical 8D arrays directly.
   Converters from current BrightEyes-MCS files may reshape the flat source
   sample axis only when:

     source_flat_bins == circular_repetition_count
                         * circular_point_count
                         * per_point_timebins

   If this is not exactly true, keep the source array under
   /views/flat_sample/source_* and add a metadata_warnings_json entry instead
   of inventing a wrong circular/time-bin split.

3. Do not create a separate mandatory data_dfd dataset.
   DFD is described by /metadata.attrs.acquisition_mode = "dfd" and by the
   digital_time_ns axis. The digital counts remain /data/primary.

4. Always include analog support in the schema.
   If analog FIFO is inactive, set analog_fifo_active = False and omit
   /data/analog or leave data_analog_path = "".

5. Digital and analog time axes may differ.
   Current BrightEyes-MCS can divide digital bins by clk_multiplier while
   analog bins are not divided. Store /axes/digital_time_ns and
   /axes/analog_time_ns separately.

6. Store raw register dictionaries unchanged.
   The normalized metadata should be readable, but raw GUI/FPGA/SPAD metadata
   must remain available under /metadata/raw.

7. Store structured attrs as JSON strings with *_json names.

8. Use variable-length UTF-8 string datasets for text arrays.

9. Record normalization conflicts in metadata_warnings_json.
